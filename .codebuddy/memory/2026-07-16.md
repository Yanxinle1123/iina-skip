# 2026-07-16 工作日志

## GPU 硬件解码实验 — 已移除

### 经过
1. 添加了 `-hwaccel videotoolbox` GPU 硬件解码
2. 用户反馈比纯 CPU 更慢
3. 添加了 `checkHwaccelSupport` 检测 + 并发限制 4
4. 用户反馈仍然比纯 CPU 慢
5. **最终决定：完全移除 GPU 解码代码**

### 为什么 GPU 在这个场景下更慢
- 帧提取参数：`fps=2, scale=9:8, format=gray` — 极低带宽
- VideoToolbox 解码后帧在 GPU 显存，滤镜处理需 GPU→CPU 内存拷贝，每帧都有开销
- VideoToolbox session 初始化比软解更重
- 对于 2fps/72字节的场景，CPU 软解 + 零拷贝反而最快

### 最终状态
- `extractFrames` 恢复为纯 CPU 软解，全并行（无并发限制）
- 磁盘缓存保留（首次 ~60s，之后 ~0.1s）
- `checkHwaccelSupport`、`extractFramesRaw`、`runWithConcurrencyLimit` 全部移除
